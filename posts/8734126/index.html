¿Que es AppArmor y porque lo necesito?

Definicion de Wikipedia: AppArmor (“Application Armor”) es un módulo de seguridad del kernel Linux que permite al administrador del sistema restringir las capacidades de un programa. Para definir las restricciones asocia a cada programa un perfil de seguridad.

¿Pero para que lo necesitaria yo, si no soy Sysadmin?

AppArmor ayuda a prevenir que las aplicaciones accedan a recursos del sistema que no necesitan para su funcionamiento normal, lo cual puede limitar el daño potencial en caso de que una aplicación se vea comprometida por un ataque malicioso.

Archlinux no instala por defecto este módulo, por lo que debes hacerlo de forma manual, necesitarás tocar archivos de configuración del gestor de arranque, entre otros. Por lo que si vas a realizar esta instalación será conveniente hacer un respaldo de los archivos importantes mediante un cp archivo archivo.backup.

1. Instalar y habilitar apparmor 

1
2

sudo pacman -S apparmor
sudo systemctl enable --now apparmor.service

2. Comprobar si apparmor figura en el parametro lsm del kernel 

1

zgrep CONFIG_LSM= /proc/config.gz

Otra manera de comprobarlo sería con el siguiente comando:

1

cat /sys/kernel/security/lsm

Deberíamos obtener una salida como esta: CONFIG_LSM="landlock,lockdown,yama,integrity,bpf" o en el caso del segundo comando, capability,landlock,lockdown,yama,bpf%

Según la wiki de Archlinux capability deberia ser omitido por lsm=, pero siempre es incluido de forma automática.

  • Evidentemente, no obtendremos apparmor en la salida debido a que aún no hemos editado el archivo de configuración de grub para añadir una modificación al parámetro lsm del kernel, e indicarle el orden de inicialización de los módulos de seguridad de Linux

3. Modificar el gestor de arranque

El archivo a modificar se encuentra en la ruta /etc/default se trata del archivo grub. En el podemos modificar desde la entrada por defecto a la hora de arrancar el equipo, hasta la resolución de pantalla del gestor de arranque. Pero he de suponer que esto ya lo sabíais.

La línea en cuestión que debemos modificar es: GRUB_CMDLINE_LINUX_DEFAULT="loglevel=3" que evidentemente tendrá el aspecto citado, en caso de no haber sido modificada y se encontrará en la línea 6 del archivo.

1
2
3
4
5
6
7
8
9

sudo vim /etc/default/grub

# Una vez dentro del archivo modificamos la línea para que se muestre del siguiente modo:

GRUB_CMDLINE_LINUX_DEFAULT="loglevel=3 lsm=landlock,lockdown,yama,integrity,apparmor,bpf"

# Una vez hecha la modificación guardamos los cambios y ejecutamos el siguiente comando:

sudo grub-mkconfig -o /boot/grub/grub.cfg

4. Comprobar el estado de Apparmor 

1
2
3
4

# La salida que obtendremos será:

Para que Apparmor entre en funcionamiento debemos reiniciar el equipo en este punto. Una vez reiniciado el equipo la salida del comando aa-enabled debería ser Si, pero por alguna extraña razón se mostrará S?. Lo que significará que el módulo apparmor se ha cargado durante el arranque.

5. Audit

El demonio de notificaciones muestra notificaciones de escritorio cada vez que AppArmor deniega el acceso a un programa. Para iniciar automáticamente el demonio aa-notify al iniciar sesión, sigue los pasos a continuación:

1
2
3
4
5
6
7

# Debemos volver a editar el archivo de configuración de GRUB, la línea que habiamos editado anteriormente en el paso 3.

sudo vim /etc/default/grub

# Añadimos el parámentro audit=1, quedando la línea del siguiente modo:

GRUB_CMDLINE_LINUX_DEFAULT="loglevel=3 lsm=landlock,lockdown,yama,integrity,apparmor,bpf audit=1"

El paquete audit es requerido por dbus, systemd... entre otros, por lo que en una instalación habitual de Archlinux se instala por defecto. Lo único que debemos hacer es habilitar su inicio durante el arranque:

1
2
3

sudo systemctl enable --now auditd.service

# Con el parámetro --now realizamos a la vez un start.

Creamos el grupo audit y añadimos nuestro usuario.

1

sudo groupadd -r audit && sudo gpasswd -a $USER audit

En el siguiente paso debes editar el archivo de configuración de audit.

1
2
3
4
5

sudo vim /etc/audit/auditd.conf

# Buscamos la línea 'log_group = root' y la modificamos para que quede del siguiente modo:

log_group = audit

# Ahora crearemos un desktop launcher, asegúrate de que la ruta ~/.config/autostart/ existe en tu directorio $HOMEdraft

touch ~/.config/autostart/apparmor-notify.desktop

vim ~/.config/autostart/apparmor-notify.desktop

# Añade el siguiente contenido al documento:

[Desktop Entry]
Type=Application
Name=AppArmor Notify
Comment=Receive on screen notifications of AppArmor denials
TryExec=aa-notify
Exec=aa-notify -p -s 1 -w 60 -f /var/log/audit/audit.log
StartupNotify=false
NoDisplay=true

Finalmente reinicia. Para comprobar que este paso se ha realizado correctamente, deberías poder ejecutar en la terminal el comando aa-notify con tu usuario si necesidad de utilizar sudo.
